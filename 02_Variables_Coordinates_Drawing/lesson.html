ü§î What is a Variable?

A variable is like a labeled box where you can store information. You give the box a name (the variable name) and put something inside it (the value). You can look inside the box later, or even change what's inside it.

üñºÔ∏è Visual Metaphor:

Imagine you have a box labeled "PlayerScore". You open it and put the number 0 inside. Later in the game, when the player scores points, you can open the box and replace 0 with 100. The label on the box never changes, but the contents can!

üéÆ How We'll Use It in Our Game:

We'll use variables to store all sorts of things: the player's position on the screen, the colour of the alien, the current score, and how many lives are left. Today, we'll create variables to hold colour values and the player's coordinates.

‚úçÔ∏è Let's Build Our Code, Step-by-Step

Step 1: Define Colour Variables

Before our game loop starts, let's create some variables to hold the RGB colour codes we'll use. This makes our code much easier to read. An RGB colour is a "tuple" - a list of numbers in parentheses.

# Colours
BLACK = (0, 0, 0)
WHITE = (255, 255, 255)
PLAYER_COLOR = (66, 135, 245)

Step 2: Define Player Variables

We also need variables to control our player's size and position. The screen's coordinate system starts with (0, 0) in the TOP-LEFT corner. The 'x' value increases as you go right, and the 'y' value increases as you go down.

# Player properties
player_width = 50
player_height = 50
player_x = (screen_width - player_width) / 2
player_y = screen_height - player_height - 10

Step 3: Fill the Background

Inside the game loop, the first thing we should do is draw the background. This covers up everything from the previous frame. We use screen.fill() and give it the colour we want.

# Inside the 'while running:' loop
screen.fill(BLACK)

Step 4: Draw the Player

Now, let's draw the player! We'll use pygame.draw.rect(). This function needs to know: where to draw (the screen), what colour to use (PLAYER_COLOR), and the position and size of the rectangle. We package the position (x, y) and size (width, height) into a pygame.Rect object.

# Inside the loop, after filling the screen
player_rect = pygame.Rect(player_x, player_y, player_width, player_height)
pygame.draw.rect(screen, PLAYER_COLOR, player_rect)

Step 5: Update the Display

All our drawing so far has happened on a hidden, off-screen canvas. To make it visible, we need one final command at the end of our game loop: pygame.display.flip(). This "flips" the hidden canvas onto the main display, showing all our new drawings.

# At the very end of the 'while running:' loop
pygame.display.flip()

‚úÖ Full Code for This Activity:

Here is the complete code. Notice how it builds on last lesson's code. Copy this into your game_activity_02.py file.

import pygame

pygame.init()

screen_width = 800
screen_height = 600
screen = pygame.display.set_mode((screen_width, screen_height))
pygame.display.set_caption("Alien Arcade - Activity 2")

# --- NEW: Define Variables ---
# Colours (R, G, B)
BLACK = (0, 0, 0)
WHITE = (255, 255, 255)
PLAYER_COLOR = (66, 135, 245) # A nice blue

# Player properties
player_width = 50
player_height = 50
# Start the player in the bottom-middle of the screen
player_x = (screen_width - player_width) / 2
player_y = screen_height - player_height - 10 # 10 pixels from the bottom

running = True
while running:
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False

    # --- NEW: Drawing ---
    # 1. Fill the background
    screen.fill(BLACK)

    # 2. Draw the player rectangle
    player_rect = pygame.Rect(player_x, player_y, player_width, player_height)
    pygame.draw.rect(screen, PLAYER_COLOR, player_rect)

    # 3. Update the display to show new drawings
    pygame.display.flip()

pygame.quit()
